# TX-Partner Kommunikations-Prinzipien

Dieses Dokument definiert die verbindlichen Kommunikationsrichtlinien für alle Content-Aktivitäten von TX-Partner. Es ist die zentrale Referenz für das Content Team und alle, die im Namen von TX-Partner kommunizieren.

**Stand:** Februar 2026
**Freigabe:** Robert Thorn, Johannes Anderl

---

## 1. Markenidentität

### Wer wir sind

```
UNTERNEHMEN:    TX-Partner (Solvum GmbH i.G.)
SLOGAN:         "Krypto-Dokumentation einfach richtig."
CHARAKTER:      "Der versierte Freund"
```

### Positionierung

**In einem Satz:**
> Wir sind Experten für Krypto-Dokumentation bei komplexen Fällen.

**Unser "Warum":**
> Krypto-Dokumentation bekommt nicht die Aufmerksamkeit, die sie verdient. Die meisten Krypto-Investoren haben kein Steuer-Problem – sie haben ein Daten-Problem in ihrem Krypto-Steuertool. Wir lenken den Fokus dorthin, wo er hingehört.

**Unsere Expertise:**
- Spezialisiert auf komplexe Portfolios (DeFi, Multi-Chain, 1000+ Transaktionen)
- Experten für den richtigen Umgang mit Krypto-Steuertools
- Korrekte Dokumentation für Steuern, Compliance und Portfolio Management

### Was wir NICHT sind

| Wir sind NICHT | Stattdessen sind wir |
|----------------|---------------------|
| Steuerberater | Pre-Tax Dokumentations-Experten |
| Steuertool-Entwickler | Tool-agnostische Spezialisten |
| Finanzberater / Vermögensverwalter | Daten-Aufbereiter |

## 2. Zielgruppen

### Priorisierung

```
PRIO 1: B2C – Private Krypto-Investoren
PRIO 2: B2B – Steuerberatungsunternehmen
PRIO 3: B2B – Anwälte / Banken (Compliance)
```

### Zielgruppen-Profile

#### Prio 1: Private Krypto-Investoren (B2C)

```
PROFIL:
├── Viel Krypto-Vermögen, vor allem On-Chain-Aktivität
├── Nutzen bereits ein Krypto-Steuertool (Blockpit, CoinTracking, etc.)
├── Haben Probleme: Fehler, Unlabeled, Missing History, negative Bestände
├── Fühlen sich überfordert, haben Angst vor dem Finanzamt
└── Suchen jemanden, der das Problem für sie löst

PAIN POINTS (dürfen angesprochen werden):
├── "Ich habe hunderte Fehler und weiß nicht, wo ich anfangen soll"
├── "Mein Tool zeigt Unsinn an, aber ich verstehe nicht warum"
├── "Ich kann nicht schlafen, weil mich das Thema belastet"
├── "Die Steuer-Deadline kommt und ich bin nicht vorbereitet"
└── "Ich habe Angst, dass das Finanzamt Probleme macht"

ANSPRACHE:
├── Du-Form
├── Emotional erlaubt
├── Pain Points direkt ansprechen
└── Charakter: "Der versierte Freund"
```

#### Prio 2: Steuerberatungsunternehmen (B2B)

```
PROFIL:
├── Haben Mandanten mit Krypto-Portfolios
├── Keine Zeit/Expertise für Krypto-Dokumentation
├── Brauchen zuverlässige, verwendbare Datengrundlage
└── Suchen professionellen Partner für Zuarbeit

ANSPRACHE:
├── Sie-Form
├── Sachlich, professionell
├── Fachjargon erlaubt
└── Fokus: Zuverlässigkeit, Qualität, Zeitersparnis
```

#### Prio 3: Anwälte / Banken (B2B)

```
PROFIL:
├── Compliance-Anforderungen (DAC8, etc.)
├── Brauchen belastbare Dokumentation
└── Langfristig interessante Zielgruppe

ANSPRACHE:
├── Sie-Form
├── Sachlich, formell
├── Fachjargon
└── Fokus: Compliance, Rechtssicherheit
```

## 3. Tonalität

### Übersicht nach Zielgruppe

| Aspekt | B2C (Investoren) | B2B (Steuerberater, etc.) |
|--------|------------------|---------------------------|
| **Anrede** | Du | Sie |
| **Stil** | Nahbar, freundlich | Professionell, sachlich |
| **Technisch** | Fachlich, aber verständlich | Fachjargon erlaubt |
| **Selbstbewusst** | Ja, Marktführer-Ton | Ja, Marktführer-Ton |
| **Emotional** | Pain Points erlaubt | Sachlich bleiben |
| **Charakter** | "Der versierte Freund" | Professioneller Partner |

### Der Charakter: "Der versierte Freund"

Für B2C-Kommunikation gilt:

```
WIR SIND:
├── Fachkompetent, aber nicht arrogant
├── Direkt, aber nicht schroff
├── Selbstbewusst, aber nicht überheblich
├── Nahbar, aber respektvoll
└── Der "Safe Space" für Krypto-Themen

WIR VERMITTELN:
├── "Du kannst mir vertrauen"
├── "Ich verstehe dein Problem"
├── "Ich weiß, wie man das löst"
├── "Du bist bei mir in guten Händen"
└── "Ich bin diskret und verschwiegen"
```

### Tonalität-Beispiele

**B2C – Richtig:**
```
"Dein Blockpit zeigt 500 Fehler und du weißt nicht mehr weiter?
Das kennen wir. Und wir wissen, wie man das löst."
```

**B2C – Falsch:**
```
"Unsere Dienstleistung umfasst die Korrektur von Transaktionsdaten
in diversen Krypto-Steuer-Applikationen."
```

**B2B – Richtig:**
```
"Wir liefern Ihnen eine belastbare Datengrundlage, die Sie direkt
für die Steuererklärung Ihrer Mandanten verwenden können."
```

**B2B – Falsch:**
```
"Hey! Deine Mandanten haben Krypto-Chaos? Wir fixen das für dich!"
```

## 4. Sprachliche Richtlinien

### Genderneutrale Sprache

**Wichtig:** Wir schreiben genderneutral, aber OHNE Sonderzeichen.

```
NICHT VERWENDEN:
├── Nutzer:innen
├── Investor*innen
├── KundInnen
└── Anleger/-innen

STATTDESSEN:
├── Alle, die Krypto besitzen
├── Krypto-Investoren (generisch)
├── Kundschaft / Klientel
├── Personen mit Krypto-Portfolio
└── Wer Krypto hält, ...
```

**Beispiele:**

| Nicht so | Besser |
|----------|--------|
| "Für alle Nutzer:innen" | "Für alle, die Krypto nutzen" |
| "Unsere Kund*innen" | "Unsere Kundschaft" / "Alle, die mit uns arbeiten" |
| "Jeder Anleger:in" | "Wer in Krypto investiert" |

### Anrede-Konsistenz

```
B2C (Du):
├── "dein Portfolio", "deine Transaktionen"
├── "Du bekommst...", "Wir helfen dir..."
└── Durchgängig in einem Text!

B2B (Sie):
├── "Ihr Mandant", "Ihre Kanzlei"
├── "Sie erhalten...", "Wir unterstützen Sie..."
└── Durchgängig in einem Text!

NIEMALS MISCHEN in einem Text!
```

### Fachbegriffe

```
ERLAUBT (auch B2C):
├── DeFi, Staking, Liquidity Mining
├── On-Chain, Multi-Chain
├── Unlabeled, Missing History
├── Transaktionen, Portfolio
└── → Diese Begriffe kennt unsere Zielgruppe

BEI BEDARF ERKLÄREN:
├── DAC8 → "die neue EU-Meldepflicht"
├── Pre-Tax → "Vorbereitung für die Steuererklärung"
└── 4-Säulen-Standard → kurz erläutern bei Erstnennung
```

## 5. Kernbotschaften

### Hauptbotschaft

> **"Nicht die Steuern sind das Problem. Es sind deine Daten."**

Langversion:
> "Dein Krypto-Steuertool zeigt Fehler? Das Problem sind nicht die Steuern – es sind deine Daten. Wir bringen sie in Ordnung."

### Unterstützende Botschaften

#### 1. Expertise bei Komplexität

```
BOTSCHAFT:
"Wir sind spezialisiert auf das, was Steuertools nicht können:
DeFi, Multi-Chain, tausende Transaktionen – je komplexer, desto besser."

VERWENDUNG:
├── Wenn es um komplexe Portfolios geht
├── Differenzierung von einfachen Services
└── Ansprache von DeFi-Power-Usern
```

#### 2. Tool-agnostisch

```
BOTSCHAFT:
"Egal ob Blockpit, CoinTracking oder andere – wir kennen die Tools
und wissen, wie man ihre Schwächen ausgleicht."

VERWENDUNG:
├── Wenn es um spezifische Tool-Probleme geht
├── Zeigt Breite unserer Expertise
└── Spricht Nutzer verschiedener Tools an
```

#### 3. Mehr als nur Steuern

```
BOTSCHAFT:
"Korrekte Dokumentation ist die Grundlage für alles:
Steuern, Compliance, Portfolio-Überblick, Ruhe im Kopf."

VERWENDUNG:
├── Erweiterung des Nutzen-Spektrums
├── Langfristige Positionierung
└── Ansprache von Compliance-Themen
```

### Proof Points

Womit wir unsere Aussagen belegen:

```
├── Erfahrung seit 2021 mit Krypto-Dokumentation
├── Hunderte Portfolios erfolgreich aufbereitet
├── Der 4-Säulen-Standard (eigenes Qualitäts-Framework)
├── DACH-Fokus mit lokalem Steuerrecht-Verständnis
└── Steuerberater-ready Output
```

## 6. Verbote & No-Gos

### Inhaltliche No-Gos

```
NIEMALS SAGEN:
├── "Wir beraten Sie steuerlich" → Wir sind keine Steuerberater!
├── "Garantiert korrekt / 100% fehlerfrei" → Keine Garantien
├── "Die besten / führend / einzigartig" → Keine Superlative
├── "Blockpit ist schlecht" → Kein Wettbewerber-Bashing
└── "In 24 Stunden erledigt" → Keine unrealistischen Versprechen

STATTDESSEN:
├── "Wir bereiten die Daten für deinen Steuerberater auf"
├── "Professionelle Dokumentation nach dem 4-Säulen-Standard"
├── "Spezialisiert auf komplexe Krypto-Portfolios"
├── "Wir lösen, was Tools nicht können"
└── "Wir melden uns zeitnah mit einer Einschätzung"
```

### Sprachliche No-Gos

```
VERMEIDEN:
├── Gender-Sonderzeichen (:, *, /, I)
├── Du/Sie-Mix in einem Text
├── Übermäßige Emojis (max. 1-2 pro Post, wenn passend)
├── Jugendsprache, Slang
├── Englisch-Deutsch-Mix ohne Grund
└── Passive Formulierungen wo aktiv besser passt
```

### Rechtliche No-Gos

```
VERBOTEN:
├── Kundennamen ohne explizite Zustimmung
├── Portfolio-Größen veröffentlichen
├── Spezifische Cases ohne Anonymisierung
├── Steuerliche Empfehlungen geben
└── Finanzberatung implizieren
```

## 7. Content-Formate & Kanäle

### Kanal-spezifische Anpassungen

| Kanal | Zielgruppe | Tonalität | Besonderheiten |
|-------|------------|-----------|----------------|
| Website | B2C + B2B | Je nach Seite | Formell auf B2B-Seiten |
| LinkedIn | B2B primär | Sie, sachlich | Thought Leadership |
| Twitter/X | B2C primär | Du, lockerer | Pain Points, kurz & knackig |
| Blog | B2C + B2B | Je nach Artikel | Fachlich, aber zugänglich |
| E-Mail | Je nach Empfänger | Persönlich | Immer passende Anrede |
| Google Ads | B2C primär | Du, direkt | Problem → Lösung |

### Hashtag-Strategie

```
PRIMÄR:
├── #KryptoDokumentation
├── #KryptoSteuern
└── #DACH

SEKUNDÄR:
├── #Blockpit #CoinTracking (bei Tool-spezifischem Content)
├── #DeFi #Bitcoin #Ethereum (bei themenspezifischem Content)
└── #DAC8 (bei Compliance-Content)

MAXIMUM: 5 Hashtags pro Post
```

## 8. Qualitätssicherung

### Checkliste vor Veröffentlichung

```
INHALT:
[ ] Kernbotschaft erkennbar?
[ ] Zielgruppe klar (B2C oder B2B)?
[ ] Richtige Anrede (Du oder Sie) durchgängig?
[ ] Keine Steuerberatungs-Claims?
[ ] Keine Garantie-Versprechen?
[ ] Genderneutral ohne Sonderzeichen?

TONALITÄT:
[ ] Passt zum Charakter "Der versierte Freund" (bei B2C)?
[ ] Selbstbewusst aber nicht arrogant?
[ ] Keine Wettbewerber-Angriffe?

FORMAL:
[ ] Rechtschreibung geprüft?
[ ] Keine sensiblen Kundendaten?
[ ] CTA vorhanden und klar?
```
